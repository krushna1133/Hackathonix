Architecture

How the
masking works.

A three-layer system that separates what the OS renders from what screen capture sees.

01

Window Initialization

Tauri spawns a frameless, transparent, always-on-top window. The Rust setup hook immediately retrieves the native HWND handle from the Win32 API.

src-tauri/src/lib.rs
02

Display Affinity Flag

SetWindowDisplayAffinity is called with WDA_EXCLUDEFROMCAPTURE. This tells the DWM compositor to exclude the window from all DXGI capture surfaces.

WDA_EXCLUDEFROMCAPTURE
03

Capture Isolation

Screen recording software using BitBlt, DXGI Desktop Duplication, or Windows Graphics Capture receives a blank region where NotePilot lives.

Windows 10/11 Only
04

React Frontend Renders

The WebView2-backed React UI draws over your local display normally — glassmorphism, live text, AI cards — only you can see it.

WebView2 + React
src-tauri/src/lib.rs RUST
// NotePilot — Zero-Visibility Setup
use windows::Win32::{
  Foundation::HWND,
  UI::WindowsAndMessaging::{
    SetWindowDisplayAffinity,
    WDA_EXCLUDEFROMCAPTURE
  }
};
 
pub fn setup(app: &App) -> Result<()> {
  let window = app
    .get_window("main")
    .unwrap();
 
  let hwnd = HWND(window
    .hwnd()?.0);
 
  // 🔒 Mask from screen capture
  unsafe {
    SetWindowDisplayAffinity(
      hwnd,
      WDA_EXCLUDEFROMCAPTURE
    )
  }
}

When your colleague shares their screen, or when you share yours, NotePilot simply doesn't exist in the capture feed. The Windows DWM compositor excludes it at the hardware composition level — not via tricks or hacks.

⚠ Windows 10 (v2004+) and Windows 11 only. Cross-platform masking requires per-OS API implementations.
